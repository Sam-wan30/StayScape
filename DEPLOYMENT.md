# Deploy StayScape to Render

This guide will help you deploy your StayScape application to Render.

## Prerequisites

- [Render account](https://render.com/) (free tier available)
- GitHub repository with your code
- MongoDB Atlas account
- Cloudinary account
- Mapbox account

## Step 1: Push Code to GitHub

If you haven't already, push your code to a GitHub repository:

```bash
git add .
git commit -m "Ready for Render deployment"
git push origin main
```

## Step 2: Create Render Web Service

1. Log in to [Render](https://dashboard.render.com/)
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository
4. Render will automatically detect your `render.yaml` file and use those settings
5. Click **"Create Web Service"**

## Step 3: Configure Environment Variables

After creating the web service, you need to set up the required environment variables:

1. Go to your web service in Render dashboard
2. Click **"Environment"** tab
3. Add the following environment variables:

| Variable | Value |
|----------|-------|
| `ATLASDB_URL` | Your MongoDB Atlas connection string |
| `MAP_TOKEN` | Your Mapbox public access token |
| `CLOUD_NAME` | Your Cloudinary cloud name |
| `CLOUD_API_KEY` | Your Cloudinary API key |
| `CLOUD_API_SECRET` | Your Cloudinary API secret |
| `SECRET` | Generate a random string (Render can auto-generate this) |
| `NODE_ENV` | `production` |

**Note:** The `SECRET` variable will be auto-generated by Render based on your `render.yaml` configuration.

## Step 4: Deploy

- Render will automatically deploy your application
- You can monitor the deployment progress in the Render dashboard
- Once deployed, you'll get a URL like `https://stayscape.onrender.com`

## Step 5: Verify Deployment

1. Open your Render deployment URL
2. Test the application:
   - Browse listings
   - Try to sign up/login
   - Create a listing (if you have user accounts)

## Troubleshooting

### Build Fails

- Check the build logs in Render dashboard
- Ensure all dependencies are in `package.json`
- Verify Node.js version (should be 20)

### Database Connection Issues

- Verify `ATLASDB_URL` is correct
- Ensure MongoDB Atlas allows connections from Render's IP addresses
- Check IP whitelist in MongoDB Atlas settings

### Environment Variables Not Working

- Double-check variable names match exactly (case-sensitive)
- Ensure no extra spaces in values
- Restart the service after adding environment variables

### Images Not Uploading

- Verify Cloudinary credentials are correct
- Check Cloudinary account has sufficient quota
